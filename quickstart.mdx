# Quick Start

This guide will walk you through the basic usage of Chettu.

## Basic Usage

<Steps>
  <Step title="Scan your current directory">
    The simplest way to use Chettu is to run it without any arguments:
    
    ```bash
    chettu
    ```
    
    This will:
    - Scan the current directory recursively
    - Respect `.gitignore` and `.chettuignore` files
    - Print file paths to stdout as they're processed
    - Copy the XML output to your clipboard (up to 50KB)
    
    <Note>
    Chettu automatically ignores common binary and media file types (images, videos, fonts, etc.) to keep output focused on source code and documentation.
    </Note>
  </Step>

  <Step title="Save output to a file">
    To save the XML output to a file, use the `-o` flag:
    
    ```bash
    chettu -o project.xml
    ```
    
    If the file already exists, Chettu will prompt you for confirmation before overwriting:
    
    ```
    File project.xml already exists. Overwrite? (y/N):
    ```
    
    <Tip>
    Use the `-R` flag to force overwrite without confirmation:
    ```bash
    chettu -o project.xml -R
    ```
    </Tip>
  </Step>

  <Step title="Scan a specific directory">
    To scan a directory other than the current one, use the `-d` flag:
    
    ```bash
    chettu -d /path/to/project -o output.xml
    ```
    
    You can also scan multiple directories at once:
    
    ```bash
    chettu -d ./src -d ./docs -o combined.xml
    ```
  </Step>
</Steps>

## Understanding the Output

Chettu generates XML with three main sections:

### 1. Project Name (Optional)

Set with the `-n` flag:

```bash
chettu -n "My Awesome Project" -o project.xml
```

```xml
<project>
    <project_name>My Awesome Project</project_name>
    ...
</project>
```

### 2. Source Tree

A visual representation of your file structure:

```xml
<source_tree>
    .
    ├── README.md
    ├── src
    │   ├── main.go
    │   └── utils.go
    └── docs
        └── guide.md
</source_tree>
```

### 3. File Contents

Each file's complete content with its path:

```xml
<files>
    <file path="README.md">
        # My Project
        This is the main README file.
    </file>
    <file path="src/main.go">
        package main

        import "fmt"

        func main() {
            fmt.Println("Hello, World!")
        }
    </file>
</files>
```

## Common Use Cases

<Accordion title="Provide context to an LLM">
  Generate a complete project snapshot and paste it into your LLM conversation:
  
  ```bash
  chettu -n "My Project" -d ./src
  ```
  
  The output is automatically copied to your clipboard. Just paste it into ChatGPT, Claude, or any other LLM interface.
</Accordion>

<Accordion title="Archive a specific project state">
  Create a timestamped snapshot:
  
  ```bash
  chettu -n "My Project v1.0" -o "snapshot-$(date +%Y%m%d).xml"
  ```
</Accordion>

<Accordion title="Scan with custom ignore patterns">
  Ignore additional files or directories:
  
  ```bash
  chettu -l "*.log" -l "dist" -l "node_modules" -o project.xml
  ```
  
  Or use a custom ignore file:
  
  ```bash
  chettu -f .my_custom_ignores -o project.xml
  ```
</Accordion>

<Accordion title="Analyze only specific file types">
  Reset default ignores and specify only what you want:
  
  ```bash
  chettu --reset-ignore -l "*.md" -l "*.txt" -o docs-only.xml
  ```
</Accordion>

## Advanced Options

### Clipboard Size Limit

Control the maximum size copied to clipboard (in bytes):

```bash
# Allow up to 200KB
chettu -c 200000 -o project.xml

# Disable clipboard copying
chettu -c 0 -o project.xml
```

### Force File Overwrite

Skip the confirmation prompt when overwriting existing files:

```bash
chettu -o project.xml -R
```

<Warning>
Use `-R` with caution as it will overwrite files without asking.
</Warning>

### Combine Multiple Options

Chettu supports combining multiple flags:

```bash
chettu -n "My Project" \
       -d /path/to/project \
       -l "*.log" \
       -o output.xml \
       -c 200000
```
